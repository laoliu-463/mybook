### 1. 获取列表流程

```mermaid
flowchart TB
    start1([开始]) --> list_api["GET /common/lessonStudent<br/>evaluateLog"]
    list_api --> list_check{分页参数<br/>是否存在?}
    list_check -->|是| list_yes["getList<br/>分页查询"]
    list_check -->|否| list_no["getAll<br/>查询全部"]
    list_yes --> list_mapper["Mapper执行SQL<br/>WHERE条件查询"]
    list_no --> list_mapper
    list_mapper --> list_desensitize["markMobiles<br/>手机号脱敏"]
    list_desensitize --> list_return["返回 PaginationVO<br/>或 List"]
    list_return --> end1([结束])

    classDef startEnd fill:#d3f9d8,stroke:#2f9e44,color:#000
    classDef api fill:#e7f5ff,stroke:#1971c2,color:#000

    classDef process fill:#fff4e6,stroke:#e67700,color:#000
    classDef action fill:#ffe8cc,stroke:#d9480f,color:#000

    class start1,end1 startEnd
    class list_api api
    class list_check decision
    class list_yes,list_no,list_mapper,list_desensitize,list_return action
```

**查询条件**：lessonId、studentId、teacherId、keyword、startDate/endDate、onlyEvaluate

### 2. 点名流程（批量签到）

```mermaid
flowchart TB
    start2([开始]) --> rollcall_api["POST /common/lesson<br/>rollCallBatch"]
    rollcall_api --> rollcall_dto["接收 List<br/>LessonSignSaveDTO"]
    rollcall_dto --> rollcall_validate{状态枚举<br/>是否合法?}
    rollcall_validate -->|否| rollcall_error[返回错误:<br/>点名状态有误]
    rollcall_error --> end2([结束])
    rollcall_validate -->|是| rollcall_teacher["获取当前教师ID"]
    rollcall_teacher --> rollcall_service["lessonService<br/>.rollCallBatch"]
    rollcall_service --> rollcall_loop[遍历处理每个<br/>签到记录]
    rollcall_loop --> rollcall_check{学员记录<br/>是否已存在?}
    rollcall_check -->|已存在| rollcall_update["addRecord<br/>更新签到状态"]
    rollcall_check -->|不存在| rollcall_create["创建新<br/>LessonStudent记录"]
    rollcall_update --> rollcall_deduct
    rollcall_create --> rollcall_deduct
    rollcall_deduct{签到状态<br/>需要消课?}
    rollcall_deduct -->|是| rollcall_setting[系统设置<br/>旷课是否扣课时]
    rollcall_deduct -->|否| rollcall_save
    rollcall_setting -->|配置扣| rollcall_dec["decLessonCount<br/>扣减学员课时"]
    rollcall_setting -->|配置不扣| rollcall_save
    rollcall_dec --> rollcall_save
    rollcall_save[更新/新增<br/>签到记录]
    rollcall_save --> rollcall_return[返回成功]
    rollcall_return --> end2

    classDef startEnd fill:#d3f9d8,stroke:#2f9e44,color:#000
    classDef api fill:#e7f5ff,stroke:#1971c2,color:#000
    classDef decision fill:#ffe3e3,stroke:#c92a2a,color:#000
    classDef process fill:#fff4e6,stroke:#e67700,color:#000
    classDef action fill:#ffe8cc,stroke:#d9480f,color:#000
    classDef error fill:#ffe3e3,stroke:#c92a2a,color:#c92a2a,stroke-width:2px

    class start2,end2 startEnd
    class rollcall_api,rollcall_dto api
    class rollcall_validate,rollcall_check,rollcall_deduct decision
    class rollcall_error error
    class rollcall_teacher,rollcall_loop,rollcall_setting,rollcall_update,rollcall_create,rollcall_dec,rollcall_save,rollcall_return,rollcall_service process
```

**签到状态枚举（SignStateEnum）**：
- NONE(0) - 未签到
- NORMAL(1) - 已签到
- LATE(2) - 补签
- LEAVE(3) - 请假
- ABSENT(4) - 旷课

### 3. 停课/复课流程

```mermaid
flowchart TB
    start3([开始]) --> stop_choice{操作类型}
    stop_choice -->|停课| stop_api1["POST /common/lesson/stop"]
    stop_choice -->|复课| stop_api2["POST /common/lesson/reopen"]
    stop_api1 --> stop_ids["接收课次ID列表<br/>List<Long>"]
    stop_api2 --> stop_ids
    stop_ids --> stop_query["lessonService<br/>批量查询课次"]
    stop_query --> stop_loop[遍历每个课次]
    stop_loop --> stop_set{操作类型?}
    stop_set -->|停课| stop_stopped["设置状态<br/>= STOPPED(0)"]
    stop_set -->|复课| stop_underway["设置状态<br/>= UNDERWAY(1)"]
    stop_stopped --> stop_notice["sendLessonStateChangeNotice<br/>发送通知"]
    stop_underway --> stop_notice
    stop_notice --> stop_batch["updateBatchById<br/>批量更新"]
    stop_batch --> stop_return["返回成功/失败"]
    stop_return --> end3([结束])

    classDef startEnd fill:#d3f9d8,stroke:#2f9e44,color:#000
    classDef api fill:#e7f5ff,stroke:#1971c2,color:#000
    classDef decision fill:#ffe3e3,stroke:#c92a2a,color:#000
    classDef action fill:#ffe8cc,stroke:#d9480f,color:#000

    class start3,end3 startEnd
    class stop_choice,stop_set decision
    class stop_api1,stop_api2,stop_ids,stop_query,stop_loop,stop_stopped,stop_underway,stop_notice,stop_batch,stop_return action
```

**课次状态枚举（LessonStateEnum）**：
- STOPPED(0) - 已停课
- UNDERWAY(1) - 进行中
- COMPLETE(2) - 已结课

### 4. 还原流程（消课返还）

```mermaid
flowchart TB
    start4([开始]) --> rollback_api["POST /common/lessonStudent<br/>/rollbackCourseNum"]
    rollback_api --> rollback_ids["接收签到记录ID列表<br/>List<Long>"]
    rollback_ids --> rollback_loop[遍历校验每个ID]
    rollback_loop --> rollback_exist{记录是否<br/>存在?}
    rollback_exist -->|否| rollback_err1[抛出异常:<br/>无法还原无签到<br/>状态的记录]
    rollback_err1 --> end4_fail([失败])
    rollback_exist -->|是| rollback_course{是否有消费<br/>课程ID?}
    rollback_course -->|否| rollback_err2[抛出异常:<br/>无法还原未设置<br/>消费课程的记录]
    rollback_course -->|是| rollback_add[添加到处理列表]
    rollback_err2 --> end4_fail
    rollback_add --> rollback_process[遍历处理列表]
    rollback_process --> rollback_count["获取实扣课次数<br/>decLessonCount"]
    rollback_count --> rollback_none["设置 signState<br/>= NONE 未签到"]
    rollback_none --> rollback_clear["设置 decLessonCount<br/>= 0 清零"]
    rollback_clear --> rollback_gt0{实扣次数 > 0?}
    rollback_gt0 -->|是| rollback_restore["rollcallCancel<br/>返还课时"]
    rollback_gt0 -->|否| rollback_skip[不返还]
    rollback_restore --> rollback_log["记录课时变动日志<br/>handleAdd"]
    rollback_skip --> rollback_log
    rollback_log --> rollback_update["updateById<br/>更新记录"]
    rollback_update --> rollback_return["返回成功/失败"]
    rollback_return --> end4([结束])

    classDef startEnd fill:#d3f9d8,stroke:#2f9e44,color:#000
    classDef api fill:#e7f5ff,stroke:#1971c2,color:#000
    classDef decision fill:#ffe3e3,stroke:#c92a2a,color:#000
    classDef action fill:#ffe8cc,stroke:#d9480f,color:#000
    classDef error fill:#ffe3e3,stroke:#c92a2a,color:#c92a2a,stroke-width:2px
    classDef restore fill:#d3f9d8,stroke:#2f9e44,color:#000

    class start4,end4,end4_fail startEnd
    class rollback_api,rollback_ids,rollback_loop,rollback_exist,rollback_course,rollback_count,rollback_none,rollback_clear,rollback_gt0,rollback_restore,rollback_skip,rollback_log,rollback_update,rollback_return decision
    class rollback_err1,rollback_err2 error
```
