# 第二大脑 - 批量处理工作流

## 📂 目录结构（PARA 方法）

```
00-收集箱/          # Inbox - 待处理的原始输入
10-项目/            # Projects - 有明确目标和截止日期的项目
   └─ 00-通用交付物/
20-领域/            # Areas - 持续关注的责任领域
30-资源/            # Resources - 参考材料和学习内容
40-交付物/          # Output - 最终可交付成果
80-模板/            # Templates - 模板文件
99-归档/            # Archive - 已完成或不再活跃的内容
   ├─ 已完成项目/
   └─ 旧资料/
```

## 🔄 核心工作流

### 工作流程图

```
手动创建笔记 → 00-收集箱/
                    ↓
            [Organize] sb-organize --all
                    ↓
            PARA 目录（10/20/30/99）
                    ↓
            [Distill] 自动提取洞见
                    ↓
            [Visualize] 自动生成 Canvas
                    ↓
            增强的笔记 + 可视化
                    ↓
            [Express] sb-express（按需）
                    ↓
            40-交付物/
```

## 🚀 快速开始

### 核心命令（一键完成所有处理）

```bash
/sb-organize --all
```

**执行内容：**
1. 扫描 `00-收集箱/` 下所有 .md 文件
2. 对每个文件自动执行：
   - ✅ 归位到 PARA 目录（10/20/30/99）
   - ✅ 提取核心洞见和执行摘要
   - ✅ 生成 Canvas 可视化
3. 输出处理结果汇总

### 日常使用流程

1. **准备阶段**：将所有待处理的笔记放入 `00-收集箱/`
2. **一键处理**：运行 `/sb-organize --all`
3. **查看结果**：检查 PARA 目录下的归位结果
4. **按需输出**：使用 `/sb-express` 生成可交付成果

## 📊 Skills 说明

### 核心 Skill
- **sb-organize**: 按 PARA 方法归类笔记（自动执行完整流程）
  - 归位到正确的 PARA 目录
  - 自动调用 sb-distill 提取洞见
  - 自动调用 sb-visualize 生成 Canvas

### 辅助 Skills（按需使用）
- **sb-distill**: 提取核心洞见和执行摘要
- **sb-express**: 转化为可交付成果
- **sb-visualize**: 生成 Canvas 思维导图

### 关键参数
- `--all`: 批量处理收集箱中所有文件（主要使用方式）
- `--auto`: 单个文件自动执行完整流程

## 🎨 Canvas 颜色规范

- **红色 (1)**: 易错点、警告、待确认
- **橙色 (2)**: 应用场景、实战案例
- **黄色 (3)**: 操作步骤、SOP、代码示例
- **绿色 (4)**: 核心要点、正确做法
- **青色 (5)**: 中心主题
- **紫色 (6)**: 策略、决策模型

## 📝 PARA 归位规则

1. **10-项目/** - 能推动某个交付，下一步行动明确
2. **20-领域/** - 长期维护的能力/责任领域
3. **30-资源/** - 暂不行动但未来可能用
4. **99-归档/** - 已完成/过期/重复的内容

## 🔗 工作流自动化

执行 `sb-organize --all` 时自动完成：
1. **sb-organize** → 归位到 PARA 目录
2. **sb-distill** → 提取核心洞见（自动调用）
3. **sb-visualize** → 生成 Canvas（自动调用）

## 💡 最佳实践

### 建议的使用频率
- **每日/每周**：将新笔记放入 `00-收集箱/`
- **定期批处理**：运行 `/sb-organize --all` 统一处理
- **按需输出**：需要交付时使用 `/sb-express`

### 笔记质量要求
在放入收集箱前，确保笔记包含：
- 清晰的标题
- 基本的内容结构
- 来源信息（如有）
