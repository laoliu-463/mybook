# 技术类笔记规范

适用场景：

- 整理 JavaGuide、技术博客、面试题
- 一篇笔记回答一个问题
- 需要代码示例、代码注释、易错点、延伸链接
- 需要适配后续自动化拆分、分类、双链和主题地图

## 1. 一篇笔记只讲一个问题

- 标题必须对应一个明确问题
- 一篇笔记只回答一个核心知识点
- 原文中存在多个问题时，应拆成多篇笔记

推荐示例：

- `Java - 静态方法和实例方法有什么不同`
- `Java - JVM、JDK、JRE 有什么区别`
- `Java - 为什么说 Java 编译与解释并存`

## 2. 标题格式

统一使用：

```text
领域 - 问题
```

例如：

- `Java - 静态方法和实例方法有什么不同`
- `Java - ArrayList 和 LinkedList 有什么区别`
- `Java 并发 - synchronized 和 ReentrantLock 有什么区别`

## 5. 代码示例规范

- 代码示例尽量短，只展示一个知识点
- 代码块控制在 `10-25` 行
- 代码块内必须有关键行注释
- 代码块外必须补 `代码说明`
- 不要粘贴与当前问题无关的样板代码

## 6. 区别类问题

标题中包含以下关键词时，必须补 `## 对比`：

- 区别
- 不同
- 对比
- compare

并优先使用表格。

## 7. 易错点

技术笔记必须显式写 `## 易错点`。

写法要求：

- 只写真正容易混淆的点
- 控制在 `2-5` 条
- 尽量使用提醒式语言

例如：

- 静态方法不能直接访问实例变量
- main 是静态方法，不能直接访问未实例化对象的成员
- 不要把 ArrayList 和 Vector 混为一谈

# Java - 静态方法和实例方法有什么不同

## 一句话结论
静态方法属于类，可以通过类名直接调用；实例方法属于对象，必须创建对象后才能调用。

## 标准回答
- 静态方法使用 `static` 修饰，属于类本身
- 实例方法不使用 `static`，属于对象实例
- 静态方法可以通过类名直接调用
- 实例方法通常需要先创建对象，再通过对象调用
- 静态方法不能直接访问实例变量和实例方法
- 实例方法既可以访问实例成员，也可以访问静态成员

## 为什么
静态方法在类加载后就已经和类绑定，不依赖具体对象是否存在。
而实例方法依赖对象实例，因此可以访问对象状态，也就是实例变量。
这就是两者在调用方式和访问权限上的根本区别。

## 对比
| 维度 | 静态方法 | 实例方法 |
|---|---|---|
| 归属 | 类 | 对象 |
| 修饰符 | `static` | 无 `static` |
| 调用方式 | `类名.方法名()` | `对象.方法名()` |
| 能否访问实例变量 | 否 | 是 |
| 能否访问静态变量 | 是 | 是 |
| 典型场景 | 工具方法、公共逻辑 | 描述对象行为 |

## 代码示例
```java
class User {
    // 静态变量，属于类
    static int count = 0;

    // 实例变量，属于对象
    String name;

    // 构造方法：创建对象时初始化 name，并统计对象数量
    User(String name) {
        this.name = name;
        count++;
    }

    // 静态方法：只能直接访问静态成员
    static void printCount() {
        System.out.println("用户总数: " + count);
    }

    // 实例方法：既可以访问实例变量，也可以访问静态变量
    void printInfo() {
        System.out.println("用户名: " + name);
        System.out.println("当前总数: " + count);
    }
}
```

### 代码说明

- `count` 是静态变量，所有 `User` 对象共享
- `name` 是实例变量，每个对象各自维护
- `printCount()` 只能直接访问静态成员
- `printInfo()` 既能访问 `name`，也能访问 `count`

## 易错点

- 静态方法不能直接访问实例变量
- 在静态方法里不能直接使用 `this`
- `main` 是静态方法，不能直接访问未实例化对象的实例方法
