# Redis - 分布式锁如何实现

## 一句话结论
Redis 分布式锁通过 `SET key value NX PX timeout` 原子命令实现，确保互斥访问和自动过期。

## 标准回答
- 使用 `SET key value NX PX milliseconds` 原子命令加锁
- value 使用唯一标识（如 UUID），解锁时验证所有权
- 使用 Lua 脚本保证解锁的原子性
- 考虑 RedLock 算法提高可靠性

## 为什么
单机 Redis 存在单点故障，多机 Redis 需要 RedLock。分布式系统中多个进程可能同时访问共享资源，需要互斥机制。

## 对比
| 方案 | 优点 | 缺点 |
|---|---|---|
| SET NX | 简单 | 无自动过期 |
| SET NX PX | 支持过期 | 单点故障 |
| RedLock | 高可靠 | 实现复杂 |

## 代码示例
```python
import redis
import uuid
import time

class DistributedLock:
    def __init__(self, redis_client, lock_key, timeout=30):
        self.redis = redis_client
        self.lock_key = lock_key
        self.timeout = timeout
        self.token = str(uuid.uuid4())  # 唯一标识

    def acquire(self, blocking=True, blocking_timeout=10):
        """尝试获取锁"""
        start_time = time.time()

        while True:
            # SET key value NX PX timeout：原子操作
            # NX：键不存在才设置
            # PX：过期时间（毫秒）
            acquired = self.redis.set(
                self.lock_key,
                self.token,
                nx=True,           # 键不存在才设置
                px=self.timeout * 1000  # 过期时间
            )

            if acquired:
                return True

            if not blocking:
                return False

            # 阻塞等待
            if time.time() - start_time > blocking_timeout:
                return False
            time.sleep(0.01)

    def release(self):
        """释放锁：Lua 脚本保证原子性"""
        lua_script = """
        if redis.call("get", KEYS[1]) == ARGV[1] then
            return redis.call("del", KEYS[1])
        else
            return 0
        end
        """
        self.redis.eval(lua_script, 1, self.lock_key, self.token)
```

### 代码说明
- `token` 使用 UUID，确保只有加锁者能解锁
- `nx=True` 保证原子性，防止并发抢锁
- Lua 脚本解锁时验证 token，防止误删他人的锁

## 易错点
- 必须设置过期时间，防止死锁
- value 必须用唯一标识，解锁时验证所有权
- 解锁要用 Lua 脚本保证原子性
- 单机 Redis 有风险，生产环境建议用 RedLock 或 Redisson
