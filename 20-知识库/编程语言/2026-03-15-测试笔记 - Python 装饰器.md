# Python - 装饰器是什么

## 一句话结论
装饰器是 Python 的语法糖，允许在不修改原函数的前提下为其动态添加功能，常用于日志、权限校验、性能测量等场景。

## 标准回答
- 本质是一个函数，接收原函数作为参数
- 返回一个新的 wrapper 函数
- 使用 `@decorator` 语法糖简化调用
- 可以带参数（多层嵌套）
- 广泛用于日志、权限校验、缓存、计时

## 为什么
如果每个函数都手动添加日志、权限校验等代码，会造成大量重复。装饰器通过 AOP（面向切面编程）思想，将横切关注点与业务逻辑分离。

## 代码示例
```python
def my_decorator(func):
    """装饰器：添加前后日志"""
    def wrapper(*args, **kwargs):
        print("调用前")           # 前置逻辑
        result = func(*args, **kwargs)  # 调用原函数
        print("调用后")           # 后置逻辑
        return result
    return wrapper

@my_decorator  # 等价于 say_whee = my_decorator(say_whee)
def say_hello(name):
    print(f"Hello, {name}!")
    return f"Hello, {name}!"

# 使用
say_hello("Alice")
```

### 代码说明
- `func` 是被装饰的函数
- `wrapper` 替代原函数执行增强逻辑
- `*args, **kwargs` 保持参数透传
- `@my_decorator` 语法糖自动将函数传入装饰器

## 带参数的装饰器
```python
def repeat(times):
    """带参数的装饰器"""
    def decorator(func):
        def wrapper(*args, **kwargs):
            for _ in range(times):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(times=3)
def say_hi():
    print("Hi!")
```

## 易错点
- 装饰器会替换原函数，可能丢失原函数的元信息（函数名、文档）
- 使用 `functools.wraps` 保持元信息
- 装饰器顺序很重要（就近原则）
- 类方法装饰器需要使用 `functools.partial` 或正确处理 `self`
