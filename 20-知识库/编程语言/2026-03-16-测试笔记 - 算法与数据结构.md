# Algorithm - 快速排序是怎么实现的

## 一句话结论
快速排序采用分治思想，选择一个 pivot（基准）元素，将数组分为小于 pivot 和大于 pivot 的两部分，递归排序两部分。

## 标准回答
- 选择一个 pivot（通常选中间元素）
- 分区：小于 pivot 放左边，大于 pivot 放右边
- 递归排序左右两部分
- 平均时间复杂度 O(n log n)，最坏 O(n²)
- 空间复杂度 O(log n)

## 为什么
快速排序是原地排序（不需要额外数组），空间效率高。通过合理选择 pivot 可以避免最坏情况，平均性能优秀。

## 对比
| 算法 | 时间复杂度(平均) | 时间复杂度(最坏) | 空间复杂度 | 稳定性 |
|---|---|---|---|---|
| 快速排序 | O(n log n) | O(n²) | O(log n) | ❌ |
| 归并排序 | O(n log n) | O(n log n) | O(n) | ✅ |
| 堆排序 | O(n log n) | O(n log n) | O(1) | ❌ |

## 代码示例
```python
def quicksort(arr):
    """快速排序：原地分区"""
    if len(arr) <= 1:
        return arr

    # 选择中间元素作为 pivot
    pivot = arr[len(arr) // 2]

    # 分区：left < pivot, middle == pivot, right > pivot
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]

    # 递归排序并合并
    return quicksort(left) + middle + quicksort(right)

# 测试
arr = [3, 6, 8, 10, 1, 2, 1]
print(quicksort(arr))  # [1, 1, 2, 3, 6, 8, 10]
```

### 代码说明
- pivot 选择中间元素，避免最坏情况
- 列表推导式分区，代码简洁
- 递归合并结果

## 易错点
- pivot 选择不当会导致 O(n²) 时间复杂度
- 快速排序不稳定，相等元素顺序可能改变
- 递归深度注意栈溢出风险
- 原地版本需要额外处理分区索引
