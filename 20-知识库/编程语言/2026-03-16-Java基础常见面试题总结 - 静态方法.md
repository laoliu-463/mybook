# Java - 静态方法和实例方法有什么不同

## 一句话结论
静态方法属于类，可通过类名直接调用；实例方法属于对象，需创建对象后才能调用。静态方法不能直接访问实例成员。

## 标准回答
- 静态方法使用 `static` 修饰，属于类本身
- 实例方法不使用 `static`，属于对象实例
- 静态方法可通过 `类名.方法名()` 直接调用
- 实例方法需先创建对象，再通过 `对象.方法名()` 调用
- 静态方法不能直接访问实例变量和实例方法
- 实例方法既可以访问实例成员，也可以访问静态成员

## 为什么
静态方法在类加载后就与类绑定，不依赖具体对象是否存在。而实例方法依赖对象实例，可以访问对象状态（实例变量）。这就是两者在调用方式和访问权限上的根本区别。

## 对比
| 维度 | 静态方法 | 实例方法 |
|---|---|---|
| 归属 | 类 | 对象 |
| 修饰符 | `static` | 无 `static` |
| 调用方式 | `类名.方法名()` | `对象.方法名()` |
| 能否访问实例变量 | 否 | 是 |
| 能否访问静态变量 | 是 | 是 |
| 典型场景 | 工具方法、公共逻辑 | 描述对象行为 |

## 代码示例
```java
class User {
    // 静态变量：所有对象共享
    static int count = 0;

    // 实例变量：每个对象各自保存
    String name;

    // 构造方法：创建对象时初始化
    User(String name) {
        this.name = name;
        count++;
    }

    // 静态方法：只能直接访问静态成员
    static void printCount() {
        System.out.println("用户总数: " + count);
        // ❌ 不能直接访问实例变量 name
        // System.out.println(name); // 编译错误
    }

    // 实例方法：可以访问所有成员
    void printInfo() {
        System.out.println("用户名: " + name);    // ✅ 访问实例变量
        System.out.println("当前总数: " + count); // ✅ 访问静态变量
    }
}

public class Demo {
    public static void main(String[] args) {
        User u1 = new User("Alice");
        User u2 = new User("Bob");

        // 调用静态方法：推荐用类名调用
        User.printCount();

        // 调用实例方法：必须通过对象调用
        u1.printInfo();
    }
}
```

### 代码说明
- `count` 是静态变量，所有 `User` 对象共享这一份数据
- `name` 是实例变量，每个对象各自保存自己的值
- `printCount()` 是静态方法，只能直接访问静态成员
- `printInfo()` 是实例方法，既能访问实例变量也能访问静态变量
- `main()` 是静态方法，不能直接调用未创建对象的实例方法

## 易错点
- 静态方法不能直接访问实例变量
- 在静态方法里不能使用 `this` 关键字
- `main` 是静态方法，不能直接调用未实例化对象的实例方法
- 虽然实例方法可以访问静态成员，但不代表所有逻辑都该写成实例方法
