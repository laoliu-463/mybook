# Python - logging 模块怎么使用

## 一句话结论
Python 的 `logging` 模块是标准库提供的日志工具，支持多级别日志、可配置的输出格式和处理器。

## 标准回答
- `logging.basicConfig()` 快速配置日志
- `logging.getLogger(__name__)` 获取日志记录器
- 5 个日志级别：DEBUG < INFO < WARNING < ERROR < CRITICAL
- 支持多种输出方式：控制台、文件、邮件等

## 为什么
日志是程序运行时的重要信息来源，用于调试、监控和问题排查。硬编码的 print 语句不便于管理和控制，而 logging 模块提供了统一的日志管理方案。

## 代码示例
```python
import logging

# 配置日志：设置级别和格式
logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)

# 获取记录器
logger = logging.getLogger(__name__)

def process_data(data):
    logger.info(f"开始处理: {data}")
    try:
        result = transform(data)
        logger.debug(f"处理结果: {result}")
        return result
    except Exception as e:
        logger.error(f"处理失败: {e}")
        raise

# 使用
if __name__ == "__main__":
    process_data({"key": "value"})
```

### 代码说明
- `level=logging.INFO` 设置日志级别为 INFO，低于此级别的日志不会输出
- `%(name)s` 输出记录器名称，`%(levelname)s` 输出日志级别
- `logger.info()` 记录信息日志，`logger.error()` 记录错误日志

## 日志级别说明
| 级别 | 数值 | 用途 |
|---|---|---|
| DEBUG | 10 | 详细信息，用于调试 |
| INFO | 20 | 确认程序按预期运行 |
| WARNING | 30 | 警告，出现异常但不影响运行 |
| ERROR | 40 | 错误，严重问题 |
| CRITICAL | 50 | 致命错误 |

## 易错点
- 日志级别设置过低会导致日志过多，影响性能
- 日志中不要记录敏感信息（密码、密钥等）
- 记得在使用前配置 logging，否则可能没有输出
- 多个模块重复配置 basicConfig 只会生效第一次的配置
