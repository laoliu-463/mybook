# Deep Learning - 注意力机制是什么

## 一句话结论
注意力机制让模型"关注"输入的关键部分，为每个位置分配权重，是 Transformer 的核心组件，广泛应用于 NLP 和 CV 领域。

## 标准回答
- 核心思想：为每个输入位置分配权重
- 计算公式：`Attention(Q, K, V) = softmax(QK^T / √d_k)V`
- Q（Query）：查询向量
- K（Key）：键向量
- V（Value）：值向量
- 自注意力：每个位置关注序列中所有位置

## 为什么
传统序列模型存在信息瓶颈，长序列时难以捕捉远距离依赖。注意力机制让模型动态选择性地关注重要信息，解决长距离依赖问题。

## 对比
| 机制 | 特点 | 适用场景 |
|---|---|---|
| 硬注意力 | 取 max | 特定任务 |
| 软注意力 | 加权平均 | 通用 |
| 自注意力 | 内部交互 | Transformer |

## 代码示例
```python
import torch
import torch.nn as nn

class SelfAttention(nn.Module):
    def __init__(self, embed_size, heads):
        super(SelfAttention, self).__init__()
        self.embed_size = embed_size
        self.heads = heads
        self.head_dim = embed_size // heads

        # 断言：embed_size 必须能被 heads 整除
        assert self.head_dim * heads == embed_size

        # 线性变换
        self.values = nn.Linear(embed_size, embed_size)
        self.keys = nn.Linear(embed_size, embed_size)
        self.queries = nn.Linear(embed_size, embed_size)
        self.fc_out = nn.Linear(embed_size, embed_size)
```

### 代码说明
- 将输入投影为 Q、K、V 三个向量
- 多头注意力（Multi-Head Attention）并行计算多个注意力
- 最后通过全连接层输出

## 易错点
- Q、K、V 来自同一个输入是自注意力
- 缩放因子 `√d_k` 防止梯度消失
- 注意力权重需要 softmax 归一化
- Transformer 使用多头注意力而非单头
