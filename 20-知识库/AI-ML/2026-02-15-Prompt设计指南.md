# Prompt - 设计技巧有哪些

## 一句话结论
Prompt 越清晰具体，LLM 表现越好。使用结构化框架（角色、背景、任务、要求、示例）可显著提升输出质量。

## 标准回答
- **清晰具体**：提供明确目的、思考方向、执行策略
- **使用框架**：角色定义 + 背景说明 + 任务描述 + 输出要求 + 示例
- **分步思考**：复杂任务拆解为步骤
- **提供示例**：Few-shot 学习

## 为什么
LLM 根据 Prompt 生成回答，Prompt 质量直接决定输出效果：
- 模糊 Prompt → 随意发挥
- 清晰 Prompt → 精准回答

## 对比
| 模糊 Prompt | 清晰 Prompt |
|---|---|
| 帮我写个程序 | 用 Python 实现快速排序，要求包含测试用例和时间复杂度分析 |
| 解释一下 Java | Java 是一种面向对象编程语言，主要特点包括自动内存管理、一次编译到处运行 |

## Prompt 框架
```
1. 角色定义（Role）
2. 背景说明（Context）
3. 任务描述（Task）
4. 输出要求（Requirements）
5. 示例（Examples）
```

## 关键技巧

### 1. 明确具体
```markdown
❌ 帮我推广新产品
✅ 为"阿里云百炼"新手机设计一条微博推广帖，突出轻薄、高性能特点
```

### 2. 提供示例
```python
# Few-shot 示例
输入：苹果
输出：fruit

输入：胡萝卜
输出：vegetable

输入：米饭
输出：?
```

### 3. 分步思考
```
步骤1：分析问题
步骤2：设计方案
步骤3：实现代码
步骤4：测试验证
```

### 4. 指定格式
```markdown
请用以下格式回答：
1. 优点：...
2. 缺点：...
3. 总结：...
```

## 易错点
- Prompt 太长不一定是好事，要抓住重点
- 不要让模型"猜测"你的意图
- 复杂任务要拆分，不是给一个长 Prompt
- 提供示例比单纯描述更有效
