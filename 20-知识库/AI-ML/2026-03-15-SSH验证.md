# Linux - 如何验证当前主机身份

## 一句话结论
不要猜测主机身份，而是通过读取主机唯一标识信息（如 machine-id、hostname、IP 地址）来验证。

## 标准回答
- 查看 hostname：`hostname`、`hostname -f`、`uname -n`、`cat /etc/hostname`
- 查看网络信息：`ip addr`、`ifconfig`
- 查看唯一标识：`cat /etc/machine-id`、`dmidecode -s system-uuid`
- 查看用户和路径：`whoami`、`pwd`

## 为什么
在多台虚拟机或容器环境中，需要确认当前操作的主机是目标机器。直接"猜测"不可靠，必须读取系统唯一标识。

## 代码示例
```bash
# 主机名
hostname
hostname -f
uname -n
cat /etc/hostname

# IP 地址
ip addr
ifconfig -a

# 唯一标识（最可靠）
cat /etc/machine-id           # Linux 唯一机器 ID
dmidecode -s system-uuid     # BIOS/UEFI UUID
lsblk                         # 块设备信息

# 当前用户和路径
whoami
pwd

# 硬件信息
uname -a                      # 系统信息
cat /proc/cpuinfo            # CPU 信息
```

## machine-id 说明
`/etc/machine-id` 是 Linux 系统启动时生成的 32 位唯一标识，重启保持不变，是验证主机身份的最可靠方式。

## 易错点
- hostname 可以随意修改，不可靠
- IP 地址可能变化（DHCP）
- machine-id 最可靠，但容器中可能相同
- 虚拟机和宿主要区分开
