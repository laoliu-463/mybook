# TCP - 超时重传机制是什么

## 一句话结论
TCP 通过超时重传和快重传保证可靠传输。RTO 动态计算，SACK 选择性确认提升重传效率。

## 标准回答
- **超时重传**：RTO 定时器到期触发，保底机制
- **快重传**：收到 3 个重复 ACK，快速响应
- **RTO**：重传超时时间，动态计算
- **SACK**：选择性确认，精确指示丢失报文

## 为什么
网络中存在丢包风险，TCP 通过重传机制确保数据可靠到达：
- 超时重传：最基础的保底机制
- 快重传：比超时更快响应
- SACK：减少不必要重传

## 对比
| 机制 | 触发条件 | 特点 |
|---|---|---|
| 超时重传 | RTO 定时器到期 | 保底，延迟高 |
| 快重传 | 3 个重复 ACK | 快速响应 |
| SACK | 支持 SACK 选项 | 精确重传 |

## RTO 计算
```python
# 简化理解
RTT = 当前采样往返时间
RTTVAR = 平滑后的偏差
RTO = RTT + 4 * RTTVAR

# 超时后 RTO 会翻倍（退避）
RTO = RTO * 2
```

## 快重传流程
```
发送: 1, 2, 3, 4, 5
接收: 1, 2, [丢失] 4, 5

接收方发送 3 个重复 ACK:
ACK 3
ACK 3
ACK 3

发送方收到 3 个重复 ACK，立即重传 3
```

## SACK
```
普通 ACK 只能确认连续数据
SACK 可以精确指示收到的非连续块

SACK 选项:
[1-3] [5-7]  表示收到 1-3 和 5-7，丢失 4
```

## 易错点
- RTO 动态计算，不是固定值
- 超时后 RTO 会指数退避
- 快重传比超时重传更快
- SACK 需要双方支持
