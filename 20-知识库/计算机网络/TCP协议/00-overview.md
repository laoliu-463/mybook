## TL;DR

- TCP 是面向连接的、可靠的、基于字节流的传输层协议
- 核心机制：三次握手、四次挥手、滑动窗口、拥塞控制、超时重传
- 面试高频：握手挥手状态机、TIME_WAIT、拥塞控制算法、粘包拆包

## 核心概念速查

| 概念 | 一句话解释 |
|------|-----------|
| MSS | 最大报文段长度，不含 TCP/IP 首部 |
| RTT | 往返时延，影响超时重传计算 |
| cwnd | 拥塞窗口，发送方维护 |
| rwnd | 接收窗口，接收方通告 |
| ssthresh | 慢启动阈值，区分慢启动和拥塞避免 |
| SACK | 选择性确认，提升重传效率 |
