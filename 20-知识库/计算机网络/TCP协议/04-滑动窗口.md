# TCP - 滑动窗口机制是什么

## 一句话结论
滑动窗口实现 TCP 流量控制，通过接收方通告的 rwnd 字段控制发送速率，防止发送方压垮接收方。

## 标准回答
- **作用**：流量控制，防止发送方压垮接收方
- **rwnd**：接收方通告的接收窗口大小
- **cwnd**：拥塞窗口（发送方估算）
- **实际发送窗口 = min(cwnd, rwnd)**

## 为什么
接收方处理能力有限，如果发送方无限发包会导致：
- 缓冲区溢出
- 包丢失
- 资源浪费

滑动窗口通过"窗口"机制实现按需发送。

## 对比
| 控制类型 | 目的 | 控制变量 |
|---|---|---|
| 流量控制 | 防止压垮接收方 | rwnd |
| 拥塞控制 | 防止压垮网络 | cwnd |

## 发送方窗口结构
```
已发送已确认  |  已发送未确认   |  可发送  |  不可发送
  [已完成]    |   [在途数据]   | [可用窗口]| [超出窗口]
     ↑              ↑             ↑           ↑
   SND.NXT    SND.UNA      SND.NXT+SND.WND
```

## 滑动过程
```python
# 简化理解
window_size = rwnd  # 接收方通告

while True:
    # 发送窗口内的数据
    send_data_in_window()

    # 收到 ACK 后滑动
    if ack_received:
        window_slide_forward()  # 窗口向前滑动
```

## 易错点
- 流量控制和拥塞控制是不同概念
- 实际窗口 = min(cwnd, rwnd)
- 窗口大小可以动态调整
- 零窗口时发送方会发送探测包
