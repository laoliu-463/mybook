# HTTP 缓存机制学习

## TL;DR

- HTTP 缓存机制学习
- 什么是 HTTP 缓存
- HTTP 缓存是一种机制，用于存储 Web 资源的副本，以便在后续请求中更快地获取这些资源。

## 什么是 HTTP 缓存

HTTP 缓存是一种机制，用于存储 Web 资源的副本，以便在后续请求中更快地获取这些资源。

## 缓存分类

### 1. 私有缓存

浏览器缓存，仅针对单个用户。

### 2. 共享缓存

代理服务器缓存，多个用户共享。

## 缓存控制

### Cache-Control 头

```http
Cache-Control: max-age=3600, public
Cache-Control: no-cache
Cache-Control: no-store
```

### ETag

实体标签，用于条件请求。

### Last-Modified

最后修改时间。

## 缓存策略

1. **新鲜度检测** - 检查是否过期
2. **再验证** - 与服务器确认
3. **返回缓存** - 返回缓存副本
