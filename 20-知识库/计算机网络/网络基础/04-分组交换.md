# Network - 分组交换是什么

## 一句话结论
分组交换将数据切分成小包独立传输，到达后重组。相对电路交换更高效，是互联网的核心传输机制。

## 标准回答
- **分组交换**：数据切分小包，独立路由，到达重组
- **电路交换**：独占专用线路，资源浪费
- **优点**：高效利用资源、抗毁性强
- **缺点**：延迟不确定、可能丢包

## 为什么
互联网需要服务海量用户，资源有限：
- 分组交换：共享带宽，资源利用率高
- 某节点故障可绕路，可靠性强

## 对比
| 特性 | 电路交换 | 分组交换 |
|---|---|---|
| 连接 | 专用通道 | 无连接/虚电路 |
| 资源 | 独占 | 共享 |
| 延迟 | 确定 | 不确定 |
| 效率 | 低 | 高 |
| 抗毁性 | 差 | 好 |
| 适用 | 电话/实时语音 | 数据/互联网 |

## 分组交换流程
```
1. 数据切分成多个分组
2. 每个分组独立选择路由
3. 经过多个路由器
4. 到达目的地
5. 按序号重组数据
```

## 分组丢失原因
- 路由器缓冲区满
- 路由切换
- 传输错误

## 易错点
- 分组交换不保证顺序
- 延迟是抖动的
- 丢包需要重传
- 现代网络混合使用
