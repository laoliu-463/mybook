# Network - 子网划分是什么

## 一句话结论
子网划分通过子网掩码将 IP 地址划分为网络位和主机位，实现 IP 高效利用和广播域隔离。

## 标准回答
- **子网掩码**：区分网络位和主机位
- **CIDR**：无类域间路由，如 /24
- **可用主机** = 2^n - 2（减去网络地址和广播地址）
- **目的**：IP 高效利用、隔离广播域

## 为什么
- IPv4 地址有限，需要高效分配
- 隔离广播域，减少网络拥塞
- 提高网络安全性

## IP 地址结构
```
IPv4: 32 位 = 网络位 + 主机位

例如 192.168.1.10/24
- 网络位: 24 位 (192.168.1)
- 主机位: 8 位 (10)
- 子网掩码: 255.255.255.0
```

## 常用 CIDR
| CIDR | 子网掩码 | 可用主机 |
|---|---|---|
| /24 | 255.255.255.0 | 254 |
| /25 | 255.255.255.128 | 126 |
| /26 | 255.255.255.192 | 62 |
| /27 | 255.255.255.224 | 30 |
| /28 | 255.255.255.240 | 14 |

## 计算示例
```
192.168.1.0/26
- 总地址: 2^6 = 64
- 可用主机: 64 - 2 = 62
- 网络地址: 192.168.1.0
- 广播地址: 192.168.1.63
- 可用范围: 192.168.1.1 - 192.168.1.62
```

## 易错点
- 子网掩码越长，主机越少
- 网络地址和广播地址不能分配给主机
- /32 只有 1 个地址，用于路由
- CIDR 替代了传统的 A/B/C 类
