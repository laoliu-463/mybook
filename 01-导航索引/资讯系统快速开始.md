# 📰 资讯系统快速开始指南

> **5 分钟上手 PKM v2.1 资讯造血系统**

---

## 🎯 系统能做什么

将每日来自 **GitHub / YouTube / Hugging Face** 的技术资讯，自动整理成：
1. **可审阅的候选包**（你决定哪些值得关注）
2. **正式日报**（每日技术动态摘要）
3. **知识卡草稿**（可深度加工的知识点）

**关键特性**：
- ✅ 人工 Review 控制质量
- ✅ 白名单信息源（避免垃圾信息）
- ✅ 反幻觉约束（只用官方摘要，不脑补）
- ✅ 强制溯源（每条都有链接）

---

## 🚀 5 步开始

### Step 1：配置信息源（首次使用，5 分钟）

文件路径：`.claude/skills/news-note-assistant/sources.yaml`

**当前默认配置**已针对你的领域（Java 后端 + Redis + MySQL + hmdp）优化，可以直接使用。

如果想自定义，修改：
- `github.repositories` - 添加关注的仓库
- `youtube.channels` - 添加关注的频道
- `huggingface.tags` - 添加关注的标签

### Step 2：生成候选包（每日，1 分钟）

对 Claude 说：
```
生成今日候选包（window_hours=24）
```

候选包会生成到：
```
00-收集箱/News-Inbox/YYYY-MM-DD/00-candidate-pack.md
```

### Step 3：Review 候选包（每日，5-10 分钟）

打开候选包，快速浏览 Top 10 事件，勾选：
- `[x] approve` - 值得关注，发布到日报
- `[x] reject` - 不感兴趣，丢弃
- `[x] needs-verify` - 需要验证真实性

**建议**：每天只 approve 3-5 条核心事件即可。

### Step 4：发布日报（每日，1 分钟）

对 Claude 说：
```
我已 review，请发布
```

系统自动生成：
- 日报 → `02-学习记录/01-日报/YYYY-MM-DD-资讯日报.md`
- 知识卡草稿 → `00-收集箱/News-Knowledge-Drafts/*.md`

### Step 5：深度加工（每周，选择性）

当积累了几张草稿卡后，选择 1-2 张深度加工：

对 Claude 说：
```
把这张草稿卡加工成正式知识卡并挂载 MOC
```

Claude 会引导你使用现有 Skills：
```
knowledge-extract → moc-index → output-crafter
```

## 📅 日常节奏建议

### 每日（10 分钟）
1. 早上生成候选包（1 分钟）
2. 通勤时 Review（5-10 分钟）
3. 发布日报（1 分钟）

### 每周（30 分钟）
1. 选择 2 张草稿卡深度加工（20 分钟）
2. 检查 News-MOC 统计（5 分钟）
3. 调整信息源配置（5 分钟）

### 每月（1 小时）
1. 回顾所有日报，提取主题趋势
2. 产出 1 篇技术文章或深度总结
3. 归档旧日报

## 💡 使用技巧

### 技巧 1：善用 reject
不是所有信息都值得关注。大胆 reject，只留下真正对你有价值的。

### 技巧 2：分批加工草稿卡
不要急着加工所有草稿卡。每周选 1-2 张最有价值的深度加工即可。

### 技巧 3：调整信息源权重
如果某个来源质量不高，在 `sources.yaml` 中降低其 priority 或直接移除。

### 技巧 4：结合项目实战
关注与当前项目（如 hmdp）相关的技术动态，直接转化为项目笔记。

## 🆘 遇到问题？

### 问题 1：候选包为空
**可能原因**：window_hours 太短，或配置的信息源近期无更新

**解决**：
- 增大 window_hours（如 48 或 72）
- 检查 `sources.yaml` 配置是否正确

### 问题 2：候选包质量低
**可能原因**：信息源与你的领域不匹配

**解决**：
- 调整 `sources.yaml` 中的仓库/频道/标签
- 提高 `ranking_weights.topic_relevance` 权重

### 问题 3：草稿卡加工后质量不高
**可能原因**：原始来源信息过于简略

**解决**：
- 在 Review 时更严格筛选，只 approve 信息丰富的事件
- 使用 `knowledge-extract` 深度加工时，补充自己的理解
