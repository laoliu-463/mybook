# 输出总规则（必须严格执行）

## A. 产物目标
生成一篇可直接放入 Obsidian 的 Markdown 笔记：能复习、能背诵、能应对面试追问。

## B. 语言与术语
- 必须用工业级术语：对象实例、引用、可见性、有序性、线程安全、扩容阈值、散列扰动、链表/红黑树等
- 禁止口水词：比如“这个东西/那个玩意/大概”这类
- 结论要可验证：关键点尽量给出机制解释（文字版复盘即可）

## C. 强制结构（必须按顺序输出）
1) YAML Frontmatter（含：tags / aliases / date / status）
2) 标题：Java 八股｜{topic}
3) Mermaid 思维导图（mindmap）：底层原理 → 核心特性 → 高频面试题 → 追问链路
4) 核心概念解析（分点，短句）
5) 源码/机制复盘（文字版“执行流程”）
6) 对比表（至少 1 个）
7) 可运行 Java 示例（完整 main，可直接复制运行）
   - 关键行必须解释：为什么这样写 + 底层发生了什么
8) 面试专栏（必须包含三块）
   - ✅ 面试怎么问
   - ⚠️ 坑点/误区
   - 🔍 追问（至少 5 条，递进）
9) 一分钟背诵版（可直接背，10~15 行）
10) 面试 Checklist（用 - [ ]）

## D. Obsidian 规则
- 必须添加 3~8 个内部链接：[[相关知识点]]
- 允许用 callout（可选）：> [!tip] / > [!warning]
- 不要使用需要插件才能渲染的特殊语法（只用标准 Markdown + Mermaid）

## E. 代码规则（强制）
- Java 代码必须“一个文件可运行”
- 需要解释关键行：例如 hash、扩容、锁、volatile、CAS 等
- 不贴大段 JDK 源码，但要把关键逻辑讲清楚
