# 计算机网络：核心流程精简版

## 🤝 一、 TCP 三次握手：如何优雅地“打招呼”？
**核心目的：** 确认双方的“听力”和“说话能力”都正常，准备开始传数据。

| 步骤 | 动作 (专业术语) | 潜台词 (通俗理解) | 确认能力 |
| :--- | :--- | :--- | :--- |
| **第一步** | 客户端发送 `SYN` | “喂，你能听到我说话吗？” | 证明客户端能“说话” |
| **第二步** | 服务端回复 `SYN + ACK` | “听到了！你能听到我说话吗？” | 证明服务端能“听”且能“说” |
| **第三步** | 客户端回复 `ACK` | “听到了，那我们开始吧！” | 证明客户端能“听” |

**💡 为什么不能是两次？**
为了防止“已失效的连接请求”突然传到服务端，产生错误。必须双方都确认对方有收发能力。

## 🚀 二、 输入网址后发生了什么？ (极简版)
把这个过程看作 **“点外卖”**：

1. **查地址 (DNS 解析)**：
   - 你在手机点开“某某餐厅”，系统帮你查到它的具体经纬度（IP 地址）。
2. **确认开门 (TCP 握手)**：
   - 打个电话给餐厅：“开门吗？”“开着呢，你要点餐？”“是的，我准备下了。”
3. **下单 (HTTP 请求)**：
   - 你提交了订单：“我要一份宫保鸡丁（GET /index.html）”。
4. **出餐 (服务器处理)**：
   - 后台厨房（服务器）根据你的要求做菜，并打包。
5. **送餐 (HTTP 响应)**：
   - 外卖小哥（网络传输）把饭送到你手上，并附带小票（200 OK）。
6. **用餐 (浏览器渲染)**：
   - 你打开包装，摆好盘（浏览器把代码变成精美的网页）。

## 🛠️ 三、 必备口诀
- **IP 地址**：收件人地址。
- **MAC 地址**：网卡身份证。
- **TCP**：靠谱，不丢包（发快递）。
- **UDP**：快，丢点没事（直播）。
- **路由器**：分拣中心（跨网传输）。
- **交换机**：寝室插排（内网互连）。
