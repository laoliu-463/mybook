# 写作规范（用于保持专业、清晰、面试导向）

## 1. 面试表达优先级
优先给：
- 结论（是什么/干什么）
- 场景（什么时候用）
- 机制（怎么做到）
- 坑（容易错哪里）
- 对比（面试最爱问）

## 2. 并发类主题的强制要点
只要主题涉及并发/线程安全，必须至少覆盖：
- 可见性（volatile / happens-before）
- 原子性（CAS / 锁）
- 有序性（重排序与内存屏障）
- “为什么线程安全/不安全”的一句话证明

## 3. 代码示例注释规则
每段代码注释必须包含至少一种：
- 意图：为什么写这段
- 机制：它触发了什么规则/机制
- 坑点：如果改成 X 会怎样
- 复杂度：时间/空间/锁竞争（必要时）

## 4. Mermaid mindmap 规则（Obsidian 兼容）
- 使用 mindmap 语法（不是 flowchart）
- 缩进用空格，层级清晰
- 节点名尽量短，覆盖模块即可，不求巨细
