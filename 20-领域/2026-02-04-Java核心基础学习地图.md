# 主题全景地图：Java 核心基础 (全栈视角版)

## 1) 全局目标（我学这个为了解决什么）

- **面试目标**：拒绝死记硬背，重点应对"顺着项目经历挖掘技术问题"的考察方式【21】。
- **工程目标**：构建"高内聚低耦合"的代码结构（OOP），掌握数据存储与流转（集合/IO），理解底层运行机制（JVM/反射）。
- **必要前置知识**：计算机基础（操作系统、网络、数据结构）【24】。

## 2) 知识地图（模块分层 + 依赖关系）

在"全栈知识体系"中，Java 基础被划分为以下由浅入深的层级：

- **L1: 面向对象与语法 (必看)**

    - **OOP 三大特性**：封装（隐藏细节）、继承（复用/IS-A）、多态（扩展性/运行期绑定）【12】【14】。
    - **类图关系**：泛化、实现、聚合、组合、关联、依赖（读懂设计模式的基础）【16】【17】【18】。
    - **核心语法**：泛型、注解、异常机制【8】【22】。
- **L2: 核心机制 (进阶)**

    - **反射与代理**：Spring 等框架的基石【8】【22】。
    - **SPI 机制**：模块化与插件化的核心（如 JDBC, Dubbo）【8】【22】。
    - **序列化**：网络传输与持久化的前提【22】。
    - **值传递**：理解 Java 方法调用的本质【22】。
- **L3: 集合框架 (高频)**

    - **Collection**：ArrayList (数组), LinkedList (链表)【8】【22】。
    - **Map**：HashMap (源码必问), ConcurrentHashMap (并发安全), TreeMap【8】【22】。
    - **底层结构**：数组、链表、红黑树【22】【24】。
- **L4: 并发与新特性 (高权)**

    - **并发基础**：J.U.C 知识体系、线程池、锁机制、CAS、AQS【9】【23】。
    - **新特性**：Java 8 (Lambda, Optional, Stream) 及 Java 11/17 升级必读【10】【11】【23】。
- **L5: JVM 底层 (基石)**

    - **内存结构**：堆、栈、方法区等【11】【23】。
    - **类加载**：加载过程、类加载器、双亲委派【5】【23】。
    - **GC**：垃圾回收算法与收集器（G1, CMS, ZGC）【11】【23】。

## 3) 学习路径（0基础 → 能面试）

- **阶段1：OOP 思维建模**

    - **必学**：封装/继承/多态的定义与代码实现，UML 类图六大关系【12】【16】。
    - **输出**：能画出项目中的类图，解释 `IS-A` vs `HAS-A`。
    - **验收**：解释"为什么组合优于继承"【17】（隐含逻辑）。
- **阶段2：容器与工具掌握**

    - **必学**：ArrayList vs LinkedList，HashMap 扩容与哈希碰撞解决，泛型与反射的使用【8】【22】。
    - **输出**：手写简单的 HashMap 或动态代理 Demo。
    - **验收**：能回答"HashMap 为什么线程不安全"。
- **阶段3：底层原理突破**

    - **必学**：JVM 内存模型，GC Roots 判断，类加载双亲委派，SPI 机制【11】【22】。
    - **输出**：JVM 参数调优清单，SPI 配置文件示例。
    - **验收**：能解释 SpringBoot 自动装配原理（基于 SPI/注解）【27】。

## 4) 模块间"贯通主线"

1. **OOP** 定义了代码的骨架与交互方式【12】。
2. **泛型与注解** 为框架提供了通用性与元数据支持【8】。
3. **集合框架** 承载了业务数据的内存存储与组织【8】。
4. **IO/NIO** 负责数据在网络与磁盘间的流转【10】。
5. **并发框架** (J.U.C) 提升了多核环境下的处理效率【9】。
6. **JVM** 是上述所有代码运行的操作系统级沙箱【11】。
7. **反射与 SPI** 赋予了系统动态扩展的能力，最终支撑起 Spring/Dubbo 等上层架构【22】。

## 5) 典型面试题型地图

1. **OOP 理解**：
    - 考点：多态的实现条件（继承、重写、向上转型）。
    - 结构：定义 -> 编译时/运行时多态 -> 代码示例【14】。
2. **集合底层**：
    - 考点：HashMap 扩容机制、红黑树转换阈值。
    - 结构：结构(数组+链表+红黑树) -> Put 流程 -> 扩容因子【22】。
3. **并发安全**：
    - 考点：ConcurrentHashMap 如何保证线程安全？
    - 结构：JDK 1.7 (分段锁) vs 1.8 (CAS + Synchronized)【9】【22】。
4. **类加载**：
    - 考点：双亲委派机制及其破坏。
    - 结构：Bootstrap -> Ext -> App -> 自定义 -> 委派逻辑【11】。
5. **内存管理**：
    - 考点：JVM 内存区域划分。
    - 结构：线程私有(栈/PC) vs 共享(堆/方法区)【11】【23】。
6. **扩展机制**：
    - 考点：什么是 SPI？应用场景？
    - 结构：接口定义 -> META-INF 配置 -> ServiceLoader 加载 -> 驱动/插件场景【22】。
7. **基础特性**：
    - 考点：String 为什么不可变？反射的优缺点？
    - 结构：源码分析 (final char[]) -> 安全性/缓存 -> 性能损耗【22】。
8. **异常处理**：
    - 考点：Exception vs Error，Checked vs Unchecked。
    - 结构：Throwable 继承体系 -> 编译期/运行期区别【8】。
9. **新特性**：
    - 考点：Java 8 Stream API 或 Optional。
    - 结构：Lambda 表达式 -> 链式调用 -> 避免空指针【10】。
10. **设计模式落地**：
    - 考点：JDK 中用到的设计模式（如 IO 的装饰者模式）。
    - 结构：模式名 -> JDK 类名 -> 解决了什么问题【10】。
11. **序列化**：
    - 考点：Serializable 接口的作用，serialVersionUID。
    - 结构：标记接口 -> 版本兼容性 -> 传输/持久化【22】。
12. **语法糖**：
    - 考点：自动拆装箱、Switch 支持 String。
    - 结构：编译前代码 -> 编译后字节码 -> 底层实现【22】。

1. **HashMap 源码剖析**

    - 关联：内存泄漏、STW、G1/CMS。
    - 来源：【11】《GC - Java 垃圾回收基础知识》
4. **Java SPI 机制**
    - 重要性：理解框架（如 SpringBoot 自动装配）原理的钥匙。
    - 关联：类加载器、设计模式。
    - 来源：【22】《Java SPI 机制详解》

    - 重要性：现代 Java 开发的标配，提升代码简洁度。

> 3. 汇总了 12 类典型面试题型（从 OOP 理解到语法糖），每题都给出了"考点-结构-来源"的标准答题框架

### 关键要点
- **分层体系**：L1 面向对象（基础） → L2 核心机制（框架基石） → L3 集合框架（高频面试） → L4 并发新特性（高权重） → L5 JVM 底层（调优基础）
- **学习路径**：0 基础需三阶段递进（OOP 思维建模 → 容器工具掌握 → 底层原理突破），每阶段都有可验收的输出标准
- **贯通主线**：OOP 定义骨架 → 泛型/注解提供元数据 → 集合承载数据 → IO/NIO 流转数据 → J.U.C 提升并发 → JVM 运行沙箱 → 反射/SPI 动态扩展
- **面试题型**：12 类高频题（OOP/集合/并发/类加载/内存/SPI/异常/新特性/设计模式/序列化/语法糖），每题都有标准答题结构
- **重点模块**：HashMap 源码（必问）、AQS 锁机制（并发核心）、JVM GC（调优基础）、SPI 机制（框架原理）、Java 8+ 新特性（现代开发标配）
- **学习目标**：面试目标是"顺着项目经历挖掘技术问题"，工程目标是"高内聚低耦合 + 理解底层机制"

### 🚀 可复用资产

- [x] 操作清单 / SOP
  **Java 核心基础学习 SOP**：
  1. 阶段 1（1-2 周）：学习 OOP 三大特性 + UML 类图六大关系，输出项目类图 + IS-A/HAS-A 解释
  2. 阶段 2（2-3 周）：掌握 ArrayList/LinkedList/HashMap，手写 HashMap Demo，验收"HashMap 为什么线程不安全"
  3. 阶段 3（3-4 周）：学习 JVM 内存模型 + 类加载 + SPI，输出调优清单，验收 SpringBoot 自动装配原理
  4. 面试准备：按 12 类题型逐个击破，每题准备"定义 → 原理 → 代码示例"标准答案
  5. 深度突破：选择 HashMap 源码、AQS 锁机制、JVM GC 中的 1-2 个模块深入研究

- [x] 决策模型
  **学习优先级决策树**：
  - 面试 < 1 个月 → 优先 L3 集合框架 + L5 JVM（高频题）
  - 面试 1-3 个月 → 完整三阶段学习路径
  - 工程能力提升 → 重点 L2 核心机制（反射/SPI） + L4 并发（J.U.C）
  - 框架原理理解 → 必学 SPI 机制 + 反射代理

- [x] 常见坑点与边界
  - **坑点 1**：学习顺序错误（直接学 JVM/并发，缺乏 OOP 基础导致理解困难）
  - **坑点 2**：只记结论不理解原理（如"HashMap 线程不安全"但说不出为什么）
  - **坑点 3**：忽略模块间贯通主线（孤立学习每个知识点，无法串联成体系）
  - **坑点 4**：面试题型准备不系统（12 类题型中只准备了常见的 HashMap/JVM，忽略了 SPI/序列化等）
  - **边界条件**：本地图适合 Java 后端开发面试准备，不涵盖 Android/大数据等专用技术栈
