# sb-visualize - 可视化技能

你正在执行 /sb-visualize 或被调用 sb-visualize skill。

## 目标
为指定笔记生成 Canvas 可视化文件（.canvas），展示知识结构、核心要点和关联关系。

## 输入
- 参数1：目标文件路径（必须）
- 参数2（可选）：
  - `--type mindmap`：思维导图模式（默认）
  - `--type flow`：流程图模式
  - `--type concept`：概念关联图模式

## 执行步骤（强制）
1. 读取目标 Markdown 文件全文
2. 解析内容结构：
   - 标题（中心主题）
   - 核心要点（一级节点）
   - 子要点（二级节点）
   - 关联关系（边连接）
3. 生成 Canvas JSON 文件（同名 .canvas）
4. 使用 PARA/CODE 颜色规范

## Canvas 颜色规范
- `"1"` 红色：易错点、警告、待确认
- `"2"` 橙色：应用场景、实战案例
- `"3"` 黄色：操作步骤、SOP、代码示例
- `"4"` 绿色：核心要点、正确做法
- `"5"` 青色：中心主题
- `"6"` 紫色：策略、决策模型

## 布局规则
### 思维导图模式（默认）
```
          [要点1]        [要点2]
              \            /
               \          /
            [中心主题(青色)]
               /          \
              /            \
          [要点3]        [要点4]
```
- 中心节点：x=400, y=300（青色）
- 要点节点：围绕中心均匀分布（绿色）
- 易错点：底部区域（红色）
- 代码示例：左侧区域（黄色）

### 流程图模式
```
[步骤1] → [步骤2] → [步骤3] → [步骤4]
```
- 从左到右线性排列
- 使用箭头连接

### 概念关联图模式
```
      [核心概念]
       /   |   \
      /    |    \
  [概念A] [概念B] [概念C]
     |      |      |
  [详细1] [详细2] [详细3]
```
- 分层展示
- 使用分组框

## 输出（必须做到）
1. 在笔记同目录生成同名 .canvas 文件
2. Canvas 包含：
   - 中心节点（标题）
   - 要点节点（≤6个）
   - 连接边（展示关系）
   - 分组（如有分类）
   - 易错点节点（如有）
3. 终端输出：
   - ✅ 已生成 Canvas：<文件名>.canvas
   - 📊 节点数：X 个
   - 🔗 连接数：Y 条

## 自动集成
- `/sb-distill --with-canvas`：提取后自动生成 Canvas
- `/sb-organize --auto --canvas`：整理流程自动生成 Canvas

## 严谨性
- 只基于笔记实际内容生成节点
- 不添加笔记中不存在的信息
- 节点文本使用笔记原文
