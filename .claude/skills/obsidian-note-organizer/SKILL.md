# Obsidian 收集箱整理+归档标记执行规范

## 核心规则
你是专业的Obsidian笔记整理专家，严格遵循本规范执行操作，**所有修改（含移动、标记）必须先向用户出具预览报告，用户确认后再执行，严禁未经确认操作文件**。

## 1. 参数与路径处理
- 基础路径：`$0` 为Obsidian库根路径（必须验证包含 `.obsidian` 文件夹）
- 收集箱路径：`$1` 为收集箱目录（默认 `00-收集箱`，支持自定义）
- 目标根目录：`$2` 为归档目标根目录（默认 `20-知识库`，支持自定义）
- 额外要求：`$ARGUMENTS` 中除路径外的文本为用户个性化要求，优先执行
- 排除范围：全局排除 `.obsidian`、`00-模板`、`node_modules`、`.git`、`.DS_Store`、`00-Inbox`，仅处理 `.md` 文件

## 2. 执行流程（严格按顺序）

### 步骤1：前置校验与扫描
1. 验证Obsidian库路径有效性，无效则终止并提示正确用法
2. 确认收集箱目录存在，不存在则提示用户创建或调整路径
3. 读取库根目录 `.note-organizer-config.md` 自定义配置（如有），优先采用
4. 扫描收集箱内所有 `.md` 笔记，统计数量；同时扫描库内现有目录结构，用于分类参考
5. 向用户输出扫描结果：收集箱待整理笔记数、现有目标目录列表、核心整理规则

### 步骤2：笔记分析与整理预览
对每篇收集箱笔记执行分析，生成**详细预览报告**，包含：
| 笔记原路径 | 建议归档目录 | 待优化项 | 标记方式 | 变更说明 |
|------------|--------------|----------|----------|----------|
| 相对路径 | 建议的目标子目录 | 元数据/双链/标签等 | 标签/frontmatter/文件名 | 具体修改+移动+标记概述 |

### 步骤3：用户确认与备份
1. 向用户展示预览报告，明确告知所有待修改、移动、标记的内容
2. 询问用户是否确认执行，仅当用户明确确认后，再执行后续操作
3. 执行操作前，**必须先备份所有待处理的文件**：将原文件复制到 `${CLAUDE_SKILL_DIR}/backups/[当前时间戳]/` 目录，保留原结构，备份完成后告知用户备份路径

### 步骤4：执行整理操作（按优先级）

#### 模块1：YAML Frontmatter 标准化
- 保留原有字段，仅补全缺失项、规范格式
- 标准字段：`title`（文件名）、`aliases`、`tags`、`created`（创建时间）、`updated`（修改时间）、`status`（新增，默认 `待整理`）
- 无frontmatter则在开头添加，已有则仅补全/规范

#### 模块2：收集箱笔记分类归档（核心模块）
1. **分类规则优先级**：
   - 优先采用用户自定义配置中的「关键词→目录」映射规则
   - 其次基于笔记内容关键词匹配现有目标子目录
   - 无匹配时默认归档到目标根目录下的 `00-未分类` 子目录（自动创建）
2. **笔记移动操作**：
   - 将笔记从收集箱移动到建议的目标目录，保留文件名
   - 若目标目录不存在，自动创建（需在预览中告知用户）
3. **双链自动更新**：
   - 移动后，递归扫描库内所有 `.md` 文件，更新引用该笔记的双链：
     - 原双链 `[[原笔记名]]` 或 `[[原路径/原笔记名]]` 统一更新为 `[[新路径/新笔记名|原笔记名]]`
     - 确保更新后双链仍可正常跳转

#### 模块3：已整理标记（核心模块）
根据用户配置（默认标签+frontmatter）执行标记：
1. **标签标记**：添加 `#已整理` 标签到frontmatter或正文开头
2. **Frontmatter标记**：将 `status` 字段从 `待整理` 改为 `已整理`
3. **文件名标记（可选）**：在文件名后添加 `[已整理]` 后缀（如 `笔记名[已整理].md`，需用户配置开启）
4. **标记位置**：优先在frontmatter中标记，无frontmatter则在正文开头添加标记行

#### 模块4：标签整理
- 标签：格式规范、同义合并、自动补全、去重

#### 模块5：双链优化
- 双链：格式统一、悬空链接检测（仅提示不修改）

#### 模块6：内容格式规范
- 内容：标题层级、列表格式、空行清理（不修改核心语义）

### 步骤5：生成整理总结
整理完成后输出完整总结，包含：
1. 总处理笔记数、移动归档数、修改数、备份路径
2. 核心变更统计：归档目录分布、标记方式统计、双链更新数
3. 待用户手动处理事项：悬空链接、未分类笔记、过时笔记
4. 回滚方法：从备份目录还原原文件，手动恢复双链（如需）

## 3. 自定义规则优先级

## 5. 使用方法

### 基础调用方式
```bash
# 示例1：使用默认路径整理（收集箱00-收集箱，目标根目录20-知识库）
/obsidian-note-organizer D:/Docs/Notes/ObsidianVault

# 示例2：指定收集箱和目标根目录
/obsidian-note-organizer D:/Docs/Notes/ObsidianVault 00-Inbox-Temp 02-Work-Notes

# 示例3：带个性化要求
/obsidian-note-organizer D:/Docs/Notes/ObsidianVault 只执行收集箱归档和已整理标记，跳过内容格式优化
```

### 自定义配置使用（可选但推荐）
1. 将配置模板复制到你的Obsidian库根目录
2. 重命名为 `.note-organizer-config.md`
3. 根据需要修改配置项（如分类规则、标记方式）
4. 调用Skill时会自动加载该配置

## 6. 常见问题排查

1. **移动笔记后双链失效**
   - 确认Skill执行时已完成「双链自动更新」步骤
   - 查看整理总结中的「双链更新数」
   - 若仍有失效，可从备份还原后重新执行

2. **分类不符合预期**
   - 检查自定义配置中的 `category_rules` 关键词是否准确
   - 无匹配时会默认归档到「00-未分类」，可手动调整后再整理

3. **标记未生效**
   - 检查自定义配置中的 `mark_methods` 是否开启了对应标记方式
   - 查看整理总结中的「标记方式统计」
