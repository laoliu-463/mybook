# sb-organize - 整理技能

你正在执行 /sb-organize 或被调用 sb-organize skill。

## 输入
目标文件名或路径（通常来自 00-收集箱/）

## 归位规则（强制，按作用，不按知识点）
1) 能推动某个交付（下一步行动明确）→ 10-项目/
2) 长期维护能力/责任 → 20-领域/
3) 暂不行动但未来可能用 → 30-资源/
4) 完成/过期/重复 → 99-归档/
5) 信息不足无法判断 → 30-资源/ 并在正文添加：
   > [!WARNING] 待确认归位：信息不足

## 输出（必须做到）
1. 读取目标文件
2. 给出"建议目标路径"（至少到一级；如能识别可到二级）
3. 执行移动到目标目录
4. 更新 YAML frontmatter：status: organized
5. 终端只输出两行：
   - ✅ 已归位：<旧路径> → <新路径>
   - 👉 建议下一步：/sb-distill 或 /sb-express（按内容选择）

## 严谨性
- 无法判断就标注待确认，不要硬猜项目名
