# sb-organize - 整理技能

你正在执行 /sb-organize 或被调用 sb-organize skill。

## 输入
- 参数1：目标文件名或路径（通常来自 00-收集箱/）
- 参数2（可选）：
  - `--auto` 或 `-a`：自动执行完整流程（organize → distill）
  - `--all`：处理收集箱中的所有文件

## 归位规则（强制，按作用，不按知识点）
1) 能推动某个交付（下一步行动明确）→ 10-项目/
2) 长期维护能力/责任 → 20-领域/
3) 暂不行动但未来可能用 → 30-资源/
4) 完成/过期/重复 → 99-归档/
5) 信息不足无法判断 → 30-资源/ 并在正文添加：
   > [!WARNING] 待确认归位：信息不足

## 工作模式

### 模式1：单步模式（默认）
```
/sb-organize <文件>
```
- 仅执行归位操作
- 输出建议下一步

### 模式2：自动模式（推荐）
```
/sb-organize <文件> --auto
```
- 自动执行：归位 → 提取
- 一条命令完成整个流程

### 模式3：批量模式
```
/sb-organize --all
```
- 处理收集箱中所有文件
- 每个文件自动执行完整流程

## 输出（必须做到）

### 单步模式输出
1. 读取目标文件
2. 给出"建议目标路径"（至少到一级；如能识别可到二级）
3. 执行移动到目标目录
4. 更新 YAML frontmatter：status: organized
5. 终端输出：
   - ✅ 已归位：<旧路径> → <新路径>
   - 👉 建议下一步：/sb-distill 或 /sb-express（按内容选择）

### 自动模式输出
1. 执行归位（同上）
2. **使用 Skill tool 自动调用下一个 skill**：
   ```
   使用 Skill tool 调用：skill="sb-distill", args="<归位后的完整路径>"
   ```
3. 终端输出：
   - ✅ 已归位：<旧路径> → <新路径>
   - ✅ 已提取：<文件名>（已追加 Distill 区块）
   - 🎉 完整流程已完成

### 批量模式输出
1. 扫描 00-收集箱/ 下所有 .md 文件
2. 对每个文件执行自动模式（包含自动调用 sb-distill）
3. 输出每个文件的处理结果
4. 最后输出汇总：
   - 📊 处理完成：共 X 个文件
   - ✅ 成功：Y 个
   - ⚠️ 需确认：Z 个

## 自动调用链（重要）
当使用 `--auto` 或 `--all` 参数时，**必须**使用 Skill tool 调用下一个 skill：
- 归位完成后 → 使用 `Skill tool` 调用 `sb-distill`
- 不要使用文字说明，必须实际调用 Skill tool

## 严谨性
- 无法判断就标注待确认，不要硬猜项目名
- 批量模式遇到错误继续处理下一个，最后汇总错误
