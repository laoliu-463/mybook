# CODE Workflow Skills

这一组技能用于执行第二大脑的 CODE 流程。

## 核心原则
1. **结构化**：严格遵守 PARA 目录规范
2. **流程化**：按 CODE 四步处理信息
3. **低幻觉**：基于文件内容，事实存疑必须标注

## /sb-capture
- **Description**: 快速捕获信息到收集箱。
- **Instruction**:
  1. 读取用户输入。
  2. 读取 `80-模板/Capture-模板.md`。
  3. 在 `00-收集箱/临时记录/` 下创建一个新文件，文件名格式为 `YYYY-MM-DD-关键词.md`。
  4. 将输入内容填入模板，写入文件。
  5. 只报告："✅ 已捕获：[文件名](路径)"。

## /sb-organize
- **Description**: 将收集箱内容分诊到 PARA 目录。
- **Instruction**:
  1. 读取指定文件。
  2. 分析其内容属性（项目/责任/资源）：
     - 有明确截止日期、下一步动作 → `10-项目/`
     - 长期维护的能力领域 → `10-核心责任/`
     - 仅作参考、暂无行动 → `11-参考资源/`
  3. 移动文件到对应目录。
  4. 更新 YAML frontmatter：`status: organized`。
  5. 报告位置并给出下一步建议（如：是否需要 /sb-distill？）。

## /sb-distill
- **Description**: 对笔记进行渐进式总结。
- **Instruction**:
  1. 读取目标文件。
  2. 读取 `80-模板/Distill-模板.md`。
  3. 基于文件内容生成"核心洞见"，**追加**到文件末尾（不要覆盖原文）。
  4. **事实校验协议**：
     - 技术参数、历史数据、引用来源必须有来源支撑
     - 若无来源，添加 `> [!WARNING] 待确认：来源缺失`
  5. 报告完成状态。

## /sb-express
- **Description**: 生成交付物。
- **Instruction**:
  1. 根据用户要求的格式（如面试题、文章），重组笔记内容。
  2. 在 `30-输出作品/` 下创建新文件。
  3. 必须包含一个"检查清单"，确保交付物可用。
  4. 报告交付物路径。

## 目录规范（PARA）
```
00-收集箱/
  临时记录/          # Capture 默认落点
  待处理素材/        # 需要进一步加工的内容

10-项目/             # 短期、有截止日期
  进行中/
  01-计划/
  02-资料/
  03-过程记录/
  04-交付物/

10-核心责任/         # 长期维护的领域
  后端能力/
  学习与成长/
  职业发展/
  表达能力/

11-参考资源/         # 兴趣与储备
  课程与书籍/
  工具资料/
  案例收藏/

30-输出作品/         # 可交付成果
  面试回答/
  技术文章/
  课程报告/

80-模板/             # 模板库

99-归档/             # 已完成或归档
  已完成项目/
  旧资料/
```

## 失败判定
- 文件路径不符合 PARA 规范
- 事实陈述无来源且未标注"待确认"
- 覆盖原文（Distill 必须追加）
