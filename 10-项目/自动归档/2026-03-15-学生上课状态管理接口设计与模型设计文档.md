# 6.1 列表查询模型

## 模型名称

`StudentLessonStatusParamDTO`

## 继承关系

`extends PaginateDTO`

## 模型说明

用于获取学生上课状态列表的查询条件入参，支持分页与条件筛选。

## 字段设计

|字段名|类型|必填|说明|示例值|
|---|---|--:|---|---|
|page|Integer|否|页码|1|
|pageSize|Integer|否|每页条数|30|
|lessonId|Long|否|课次 ID|1001|
|classId|Long|否|班级 ID|2001|
|courseId|Long|否|课程 ID / 消课课程 ID|3001|
|studentId|Long|否|学员 ID|50001|
|keyword|String|否|学员姓名 / 手机号关键字|张三|
|signStateCode|Integer|否|上课状态编码|1|
|startDate|LocalDate|否|查询开始日期|2026-03-01|
|endDate|LocalDate|否|查询结束日期|2026-03-31|

## 模型设计说明

1. 若页面固定为某课次下查询，可要求：
    
    - `lessonId` 必填
        
2. 若页面支持多课次综合查询，则 `lessonId` 可为空
    
3. `keyword` 支持学员姓名 / 手机号模糊检索
    
4. 若返回手机号，必须脱敏

|---|---|--:|---|---|

# 9. 接口二：设置学生上课状态

## 9.1 接口说明

用于对前端勾选的课次学员记录批量设置目标上课状态。  
本质是课次学员级别的状态处理，而不是学员主档状态处理。

---

## 9.2 接口定义

- **接口名称**：`batchSetStatus`
    
- **请求方式**：`POST`
    
- **请求路径**：`/common/lessonStudent/batchSetStatus`
    

---

## 9.3 Controller 设计建议

```java
@ApiOperation("设置学生上课状态")
@Log(description = "设置学生上课状态", type = "学生上课状态管理")
@PostMapping("/batchSetStatus")
@PreventMultiSubmit
public JsonResponse batchSetStatus(@Valid @RequestBody BatchSetStudentLessonStatusSaveDTO dto,
                                   BindingResult result) {
    CommonUtil.handleValidMessage(result);
    lessonStudentStatusService.batchSetStatus(dto);
    return JsonResponseUtil.success();
}
```

---

## 9.4 Service 设计建议

```java
void batchSetStatus(BatchSetStudentLessonStatusSaveDTO dto);
```

---

## 9.5 处理流程

1. 接收 `lessonStudentIds`
    
2. 查询对应课次学员记录
    
3. 获取签到信息
    
4. 校验目标状态是否合法
    
5. 非法则返回错误
    
6. 获取教师 ID
    
7. 遍历处理每条记录
    
8. 判断当前记录是否存在
    
    - 存在：更新状态/签到记录
        
    - 不存在：创建状态/签到记录
        
9. 判断是否消课
    
    - 是：执行扣减课时
        
    - 否：不扣课时
        
10. 保存处理记录
    
11. 记录操作日志
    
12. 返回成功
    

---

## 9.6 流程图对齐要求

该接口必须明确体现两个关键点：

### 1）存在则更新，不存在则创建

不可只写“批量更新”。

### 2）是否消课判断

不可只写“修改状态”，必须包含：

- 是否消课
    
- 消课则扣课
    
- 不消课则不扣课
    

---

## 9.7 校验规则

- `lessonStudentIds` 非空
    
- 自动去重
    
- `targetStatusCode` 必填
    
- 状态值必须合法
    
- 已处于目标状态的数据建议拦截
    
- 越权数据不可处理

## 10.5 处理流程

2. 判断操作类型：
    
    - `STOP`
        
    - `RESUME`
