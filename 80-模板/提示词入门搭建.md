# 提示词入门搭建

## TL;DR

- 元提示词（Meta-prompting）是一种高级提示工程技术，通过让 AI 先规划再执行来提升输出质量
- 核心原则：分解任务 → 专家验证 → 迭代核验 → 降低幻觉
- 适用场景：复杂推理、代码生成、多步骤任务

---

## 背景与问题

用户通常带着不完整的想法来找 AI，不确定如何组织结构、设置约束或降低事实错误。元提示方法通过协调多位"专家"来验证和优化方案。

---

## 核心机制拆解

### 1. 元提示六大步骤

| 步骤 | 英文 | 说明 |
|------|------|------|
| 1 | Request the Topic | 索要主题，明确用户目标 |
| 2 | Refine the Task | 细化任务，确认输出形式和数据源 |
| 3 | Decompose & Assign Experts | 拆解任务，分配给不同专家 |
| 4 | Minimize Hallucination | 降低幻觉，要求验证和声明不确定性 |
| 5 | Define Output Format | 定义输出格式 |
| 6 | Generate the Prompt | 生成最终提示词 |
| 7 | Verification and Delivery | 验证与交付 |

### 2. 专家协调原则

- **新视角**：避免同一位专家既生成又验证
- **迭代验证**：对可能产生错误的任务强调迭代验证
- **反对猜测**：不确定时必须声明

---

## 常见坑与边界

> [!warning] 常见误区
> - 不要假设未验证的事实
> - 不要让同一专家同时负责生成和验证
> - 避免过度交互，给用户压迫感

---

## 输出格式模板

```markdown
[角色定义 - 简短直接，强调核验]

Context
[用户任务/目标/背景]

Instructions
[分步骤指令]

Constraints
[限制条件：时间、风格、字数]

Output Format
[精确输出结构]

Examples
[用户提供的示例]
```

---

## 最佳实践

1. 复杂任务先拆解再分配专家
2. 涉及代码/数学时调用 Expert Python
3. 不确定时要声明，不要猜测
